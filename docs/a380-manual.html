MSFS Blind Assist — FlyByWire A380X Complete Manual

A complete, screen-reader-first manual for flying the FlyByWire A380X in Microsoft Flight Simulator with MSFS Blind Assist (MSFSBA). It covers every feature MSFSBA exposes on the A380 — what it does and exactly how to use it, step by step — plus how the A380's avionics differ from a classic Airbus MCDU. Keyboard shortcuts are listed beside each feature and collected in one reference at the end. Built for total reliance on the screen reader: nothing here assumes you can see the screen.

How to read this manual. Use your screen reader's heading navigation (H, or 1–4 for heading levels) to jump between sections, and the links list to jump to the table of contents below. Every section is self-contained. Keyboard shortcuts appear like this: Ctrl+M.

1. Orientation: what this is and what you need

The FlyByWire A380X is a free, high-fidelity simulation of the Airbus A380-842 (Rolls-Royce Trent 972 engines) for Microsoft Flight Simulator, made by FlyByWire Simulations — the same team behind the A32NX. Microsoft Flight Simulator does not ship its own A380, so the FBW A380X is the de-facto A380. This manual targets the FlyByWire A380X DEVELOPMENT build (the rolling dev/master release, updated continually) — not a specific numbered stable version. It is deeply simulated but still evolving, so some features are incomplete and details may change between dev updates (see Part 14).

MSFS Blind Assist (MSFSBA) makes that aircraft flyable without sight. It reads the cockpit displays aloud, lets you operate the MFD/MCDU, flyPad, checklists and every system panel from the keyboard, and announces important changes as they happen. On the A380, MSFSBA reads the real cockpit displays live through the simulator's internal display engine (the "Coherent" debugger) — there is no add-on to install, no Community-folder mod and no Developer Mode required. You just need:

  • The FlyByWire A380X installed (via the FlyByWire installer) and loaded in the sim.
  • The cockpit displays powered (battery on, or a powered/ready preset) so there is something to read.
  • A screen reader running (NVDA is fully supported; others via Tolk; SAPI fallback otherwise).
MSFSBA detects the A380 automatically when you load it. The first time it connects there is a short silent grace period — on first detect nothing is announced (so you are not spammed with the starting cockpit state); only changes after that speak.

2. The A380 cockpit, and why it is NOT a classic MCDU

If you have flown the A320 (Fenix, FBW A32NX, or the default), you know the MCDU: one small unit per pilot that combines a screen and a hardware keypad, with six "line select keys" (LSKs) down each side. You type into a scratchpad and push the LSK next to a line to send it there.

The A380 has no MCDU. It uses Airbus's second-generation cockpit (later shared with the A350), which splits that one unit into two separate things:

The MFD (Multi-Function Display)

There are two large MFDs — one for the Captain, one for the First Officer — living on the main instrument panel. Each MFD hosts four domains: FMS (the flight-management pages — flight planning, performance, fuel), ATC COM (datalink), SURV (surveillance: TCAS, terrain, weather radar), and FCU BKUP (a backup way to drive the autoflight panel). The FMS domain is where you do almost everything.

The KCCU (Keyboard Cursor Control Unit)

On the pedestal there are two KCCUs. Each is a separate input device with a trackball + click button (a cursor), a QWERTY keyboard, and dedicated FMS jump-keys: DIR (direct-to), PERF, INIT, NAVAID, F-PLN, DEST, SEC INDEX. You move a cursor over the MFD and click fields, open dropdown menus, and pick choices — there are no LSKs.

What this means for you. MSFSBA hides all of that hardware complexity. You do not need the KCCU, the cursor, or the trackball. MSFSBA opens the MFD as a flat, screen-reader-friendly list and lets you arrow through it and press Enter — it drives the real display underneath for you. (Technically MSFSBA types directly into the focused field the same way the real cockpit keyboard does, so the KCCU "keyboard mode" does not need to be switched on.)

The key differences, summarised

So when this manual says "the MCDU", it means the A380's MFD FMS pages, presented by MSFSBA as a list you arrow through.

3. MSFSBA basics: modes, windows, and the panel tree

Hotkey modes

Most MSFSBA shortcuts live in one of two modes, so a handful of letter keys can do many things:

  • Press ] (right bracket) to enter OUTPUT mode — reading things aloud (altitude, speed, displays, distances).
  • Press [ (left bracket) to enter INPUT mode — doing things (opening the MCDU/flyPad, autopilot, teleports, taxi).
  • Escape exits the active mode. Pressing the same bracket again cancels it.
  • A few chords are direct (no bracket needed): Ctrl+M, Ctrl+E, Ctrl+Shift+B, Ctrl+1/2.

The full list is in Part 13. Throughout this manual, where a shortcut needs a mode it is noted as "(OUTPUT)" or "(INPUT)".

4. The MFD / MCDU in depth

Open it with (INPUT) Shift+M. The window shows one thing per line. The page title is announced when the page changes; the scratchpad/message line is announced when it changes.

How lines read

Actionable lines end with a role word that tells you what you can do:

Combo boxes. A closed dropdown reads, for example, "RWY, 30R, combo box, collapsed" — so you hear the field, its current selection, and that it is closed. Press Enter and it reads "expanded", with each choice listed immediately below as an "option" line. Pick one with Enter; the dropdown closes. When a combo box (or a waypoint's revision menu) is open, MSFSBA lists all of its options directly underneath the control that opened them, one per line — so you arrow down from the combo box straight into its choices (they would otherwise be scattered between other lines).

A "MCDU (Captain) / First Officer" selector chooses which side you are driving (they are independent in the real aircraft).

Moving between pages — three ways

  1. The "go to MFD page" combo at the top lists all 20 pages, grouped by tab: ACTIVE (F-PLN / PERF / FUEL & LOAD / WIND / INIT); POSITION (MONITOR / REPORT / NAVAIDS / IRS / GNSS / TIME); SEC INDEX (SEC 1–3); DATA (STATUS / WAYPOINT / NAVAID / ROUTE / AIRPORT / PRINTER).
  2. Quick page buttons (Alt+letter): Alt+I INIT, Alt+F F-PLN, Alt+P PERF, Alt+R RAD NAV, Alt+S SEC FPL, Alt+A ATC COM, Alt+D DIR, Alt+H Refresh.
  3. The MFD's own in-page tabs, which appear in the line list.

Some pages (WIND, REPORT, GNSS, TIME, the DATA sub-pages) only open once the FMS has the data they need — that is real aircraft behaviour; they read "disabled" until then.

ATC COM (datalink) and SEC flight plan

Beyond the FMS pages, the MFD's other domains are reachable as pages too:

  • ATC COM (Alt+A) is the datalink: the CONNECT page (logon/active centre), the REQUEST and REPORT pages, the message record (the CPDLC message log — when MSFSBA announces "ATC Message Waiting", open MSG RECORD to read the controller's clearance line by line), the D-ATIS list, and the emergency page. They read as line lists like the FMS pages; you arrow and press Enter on the buttons/fields the same way. This is how you read CPDLC uplinks and request/receive a digital ATIS without seeing the screen. Note: CONNECT, MSG RECORD and D-ATIS work; REQUEST, REPORT and EMERGENCY are still work-in-progress in the FlyByWire dev build and currently show "ERROR 404 NOT FOUND" — MSFSBA already navigates to them, so they will read normally once FBW implements them.
  • SEC flight plan (Alt+S) opens the SECONDARY flight plans (SEC 1–3) — the standby routes the A380 keeps for a runway change or alternate. They read exactly like the active F-PLN, so you can build and review a secondary plan and then activate it.

Reading the flight plan (F-PLN)

The F-PLN page lists the route as one line per waypoint, in order. MSFSBA folds each waypoint together with the leg that leads to it and any altitude constraint, so each line reads cleanly, for example:

BI613, via BASU1D, 5 NM, track 294°, at or below 5000 feet

  • BI613 — the waypoint name.
  • via BASU1D — the procedure or airway that leg belongs to (a SID, STAR or airway). It repeats on each leg it covers, so you always know which procedure you are in.
  • 5 NM — the leg distance from the previous waypoint, in nautical miles.
  • track 294° — the leg track (computed in flight).
  • at or below 5000 feet — an altitude constraint (decoded: + = "at or above", - = "at or below", plain = "at").
  • In flight, an ETA and speed/altitude predictions fold in as the FMS computes them.

Seeing the whole route. The page shows a window of about a dozen waypoints with the destination pinned at the bottom. The last waypoint you hear before DEST is the bottom of the current window, not the end of the route — press Ctrl+Down to scroll to the next set and Ctrl+Up to go back. Paging walks the entire route with a small overlap between windows, so nothing is skipped.

Revising the flight plan

Every waypoint line is a button. Land on it and press Enter to open that waypoint's lateral-revision menu — exactly like clicking the waypoint on the real MFD. The options then appear as a group of "option" lines; arrow to one and press Enter:

  • FROM P.POS DIR TO — direct-to this waypoint (skip everything before it).
  • INSERT NEXT WPT — insert a new waypoint after this one.
  • DELETE * — remove this waypoint.
  • HOLD — build a holding pattern here.
  • AIRWAYS — join an airway onward from here.
  • OVERFLY * — force the aircraft to fly directly over it.
  • DEPARTURE / ARRIVAL — jump to the SID/STAR editor (enabled at the ends of the route).

Disabled options read "dimmed". Some revisions stage a temporary (TMPY) plan: scroll to the bottom and press Enter on "TMPY INSERT" to commit it, or "TMPY ERASE" to cancel.

Clearing a discontinuity

A gap in the route reads as a line that simply says "DISCONTINUITY, button". The FMS inserts these where two procedure segments do not join (commonly between the end of a STAR and the approach). To clear one: land on that line, press Enter to open its menu, then arrow to "DELETE *" and press Enter — the two legs join and the discontinuity is gone. (If you would rather bridge it with a fix, open the previous waypoint instead and use "INSERT NEXT WPT".)

A discontinuity that follows a manual leg cannot — and should not — be deleted; that one is intentional.

5. Flight planning, step by step

The recommended order is: load the route (SimBrief), set the departure, set performance and weights, set the arrival, then review.

5a. Load the route from SimBrief

  1. Set your SimBrief user ID in the flyPad first (Settings), and generate an OFP on SimBrief with "Detailed Navlog" enabled (the FMS import needs it). Do not set a destination on the MSFS World Map before the flight — if you do, the aircraft imports that plan instead and the company-request option below will not appear.
  2. Open the MCDU (INPUT Shift+M) and go to INIT (Alt+I).
  3. Arrow to "CPNY F-PLN REQUEST" and press Enter. Wait until the scratchpad reads "COMPANY F-PLN RECEIVED WAITING FOR INSERTION".
  4. The button now reads "RECEIVED CPNY F-PLN". Press Enter on it; a small menu opens. Arrow to "INSERT*" and press Enter. The whole route loads — flight number, from/to, alternate, cruise level and cost index, all from SimBrief.
  5. Arrow through F-PLN to confirm the route reads as expected. If a line reads "NOT IN DATABASE" or "AWY/WPT MISMATCH", your SimBrief AIRAC cycle is older than the sim's nav data — regenerate the OFP on the matching cycle.
  6. FLT NBR, destination and ALTN stay editable afterwards if you need to change them: arrow to the field, type, Enter.

If "CPNY F-PLN REQUEST" reads "dimmed", it is because either a destination is already set on the world map or SimBrief/Navigraph is not configured — both are correct sim behaviour.

No SimBrief? Build the plan by hand in this order: on INIT type the city pair (FROM/TO) plus FLT NBR, ALTN, CI and CRZ FL; set the DEPARTURE (5b); add en-route legs by opening a waypoint and choosing AIRWAYS, entering VIA (airway) then TO (next waypoint) and repeating; set the ARRIVAL (5d); then INSERT the temporary plan. It is the same pages as the SimBrief flow, just typed in yourself.

5b. Set the departure (runway + SID)

  1. Go to F-PLN (Alt+F) and open the DEPARTURE button (or open the origin waypoint and choose DEPARTURE).
  2. Open the RWY combo, pick your departure runway.
  3. Open the SID combo, pick the departure. Leave TRANS unless your route needs one.
  4. Scroll to the bottom and press Enter on "INSERT TMPY *" to commit.

5c. Performance and weights

Open FUEL & LOAD (via INIT or the page combo) to set or confirm ZFW, ZFWCG, BLOCK fuel, TAXI, PAX and reserves — these usually come from the flyPad Dispatch page. Open PERF (Alt+P) for the takeoff, climb, cruise, descent and approach sub-tabs (V-speeds, FLEX temp, THS, transition altitude). FlyByWire's own takeoff-performance calculator (on the flyPad) is the easy way to get the takeoff numbers.

Minimums (baro MDA and radio DH) are entered on the PERF APPR tab — there is no separate panel for them, by design. MSFSBA reads them back and announces them when they change ("Baro minimum 200 feet", "Decision height 200 feet").

5d. Set the arrival (runway, approach, STAR)

  1. On F-PLN, press Ctrl+Down repeatedly to scroll to the destination airport line at the very end of the route, then press Enter on it. (The bottom "DEST" summary line is read-only — use the destination waypoint line, or its ARRIVAL revision.)
  2. Open the RWY combo, pick the landing runway. Open APPR for the approach (ILS/RNAV), STAR for the arrival, and VIA/TRANS if required. Each combo announces its current selection and its choices.
  3. Scroll down and press Enter on "INSERT TMPY *" to commit.
  4. Arrow back through F-PLN to confirm the route reads continuously into the approach. If a discontinuity appears between the STAR and the approach, clear it (see Clearing a discontinuity).

Note: the A380 always keeps a destination runway — there is no "no runway" option. To change the arrival you pick a different runway/approach; you cannot remove it entirely.

5e. Radio navigation

On RAD NAV / NAVAIDS (Alt+R), VORs auto-tune when in range; you can enter a VOR/ILS by ident or frequency. The arrival ILS auto-tunes within about 250 nm once in the climb phase.

6. The flyPad / EFB

Open it with (INPUT) Shift+T. The flyPad (flyPadOS 3) is FlyByWire's tablet. MSFSBA renders it as a real web document in an embedded browser, so your screen reader gets full browse mode: use heading navigation (H), the links list (K), and arrow-by-line; edit fields are real edit fields; page changes are announced. 7. Checklists (the live ECL)

The A380's real checklists live on the Electronic Checklist (ECL), shown on the Engine/Warning Display and driven by the ECAM Control Panel. MSFSBA opens it as a fully interactive list that always matches the real cockpit, including any abnormal procedure that an ECAM alert raises.

There are two checklist keys (both in OUTPUT mode, press ] first): Shift+C opens a simple static text checklist (cold-and-dark to shutdown), and Ctrl+Shift+C opens the live ECL described here.
  1. Open the live ECL with (OUTPUT) Ctrl+Shift+C. The checklist is shown automatically when the window opens (and hidden again when you close it) — there is nothing to toggle.
  2. Arrow Up/Down moves the real checklist cursor and reads each line, telling you whether it is checked and whether the item is manual or sensed.
  3. Press Enter: on the checklist menu this OPENS the highlighted checklist; inside a checklist it TICKS the highlighted manual item and moves on.
  4. Backspace steps back to the menu. The Abnormal procedures button lists active ECAM procedures.

Sensed items tick themselves the moment the aircraft detects the action (set the park brake and the item reads "PARK BRK ON, checked"). Abnormal procedures present automatically when a failure occurs. F5 refreshes, Escape closes.

8. Displays: PFD, ND, ISIS, System Display, E/WD

PFD (Primary Flight Display)

The PFD window reads the Flight Mode Annunciator (FMA) — the autoflight modes — plus the approach capability and any PFD messages. The FMA is the most important readout in flight; MSFSBA also announces FMA changes automatically (see Part 9). Tape values (speed, altitude, attitude) come from instruments and have their own hotkeys.

Navigation Display (ND)

Reads the ND mode (Rose ILS/VOR/Nav, Arc, Plan) and range, the active "TO" waypoint (ident, distance, bearing, ETA), cross-track error, RNP, and localizer/glideslope deviation.

Standby Instruments (ISIS)

Open with (OUTPUT) Alt+I: standby attitude (pitch and bank), magnetic heading, indicated airspeed and Mach, indicated altitude with its baro reference (STD or QNH and the setting), and the LS overlay state.

System Display (SD) — the ECAM system pages

The SD window reads any of the A380's 16 system pages: ENG, APU, BLEED, COND (air conditioning), PRESS (cabin pressure), DOOR, ELEC AC, ELEC DC, FUEL, WHEEL (wheels/brakes), HYD, F/CTL (flight controls), C/B (circuit breakers), CRZ (cruise), STATUS, VIDEO. Pick a page from the "SD page" combo and it drives the real display there, then reads it (per-tank fuel across the 11 tanks, gross weight, CG, temperatures, pressures, door states, engine parameters, and so on). If the live display is briefly unreachable it falls back to a direct sensor decode.

Reading an SD page straight from the panel (no window). The ECAM Control Panel (Pedestal) also has a System Display Page combo. Selecting a page there both switches the real SD and reads that page's decoded content aloud — so you can check any system from the panel without opening the SD window. The values read in engineering form, e.g. ENG: "N2 64.8, N3 64.1, fuel flow 810, oil quantity / temperature / pressure per engine"; BLEED: pack temperatures, bleed pressures, valve states; HYD: "GREEN 5100 PSI, YELLOW 5100 PSI", pump states. The graphical pages (Wheel/Fuel/Door) read their values too, though their on-screen schematic layout is flatter in text. Don't open the SD window and use the panel combo at the same time — they share one connection to the display.

E/WD (Engine/Warning Display)

Open with (OUTPUT) Alt+E: per-engine throttle/N1/EGT/fuel-flow, the memo lines, and the active warnings with their ECAM action items. Alt+E also speaks the current warnings immediately. ECAM warnings and memos are also announced automatically as they appear. You can toggle automatic E/WD call-outs with Ctrl+E.

The A380's Flight Warning System (FWS) is read in full and announced as it changes: the failure titles and their action lines (in red/amber/white order, exactly as the crew would read them down the display), the green memos (both columns), and the PFD memo and limitations lines. MSFSBA also speaks the small status-area reminders at the bottom of the display when they appear and again when they clear: STS ("Status message, check the status page"), ADV ("Advisory, check the system display"), and FAILURE PENDING. The display power-up safety-test ("Display safety test in progress") is announced too, so you know when the screens are booting. All of these route through the same call-out channel, so Ctrl+E mutes them together.

9. Automatic announcements (and how to mute them)

MSFSBA speaks important changes as they happen — you do not have to poll for them. Everything below is announced on change; you can silence any of it from the Monitor Manager (Ctrl+M), which is a checkbox list of every monitored item and remembers your choices.

Warnings and cautions

Master Warning, Master Caution, Autoland Warning, TCAS Fault, engine and APU fire, icing, and the overhead pushbutton FAULT lights (batteries, generators, IDGs, pumps, bleeds, packs, and more). The full E/WD failure stream — titles, action lines, memos and the STS/ADV/FAILURE-PENDING status reminders — is described in Part 8. When a failure makes the System Display automatically switch to the relevant system page, that page change is announced too (for example "System Display Page, F-CTL"), the A380 equivalent of the classic "ECAM page" call-out.

Autoflight modes (the FMA, spoken like the PFD)

  • Vertical mode: ALT, ALT*, OP CLB, OP DES, V/S, FPA, CLB, DES, FINAL, G/S, LAND, FLARE, ROLL OUT, SRS, TCAS, and so on.
  • Lateral mode: HDG, TRACK, NAV, LOC*, LOC, LAND, FLARE, ROLL OUT, RWY, GA TRK.
  • Autothrust mode: Man TOGA/Flex/MCT/Thrust, Speed, Mach, Thrust Climb, Thrust Idle, Alpha Floor, TOGA Lock, and the lever prompts (Levers Climb, Thrust Lock, etc.).
  • Approach capability: CAT 1, CAT 2, CAT 3 Single, CAT 3 Dual.
  • Flight phase: Preflight, Takeoff, Climb, Cruise, Descent, Approach, Go Around, Done. And the FCU managed/selected indicators when you push or pull a knob.

Numeric callouts

Altimeter setting on the knob and on STD↔QNH / hPa↔inHg changes; the approach minimums; thrust-lever detents ("All thrust levers Idle"); flaps and gear; and the A380's runway-overrun protection (ROW/ROP): "Runway too short", "If wet, runway too short", "Max braking", and "Runway ahead" when taxiing toward a runway.

Units: metric altitude and weight

The A380 has two unit toggles that MSFSBA honours in its own readouts, so what you hear matches the cockpit. When the FCU metric-altitude button (MTRS) is on, every altitude MSFSBA speaks is in metres — the current-altitude keys (A MSL, Q AGL), the FCU selected-altitude read (Shift+A), and the selected/standby altitude panel fields — and, when you set the FCU altitude (Ctrl+A), the value you type is taken as metres. Turn MTRS off and everything is feet again. Separately, the flyPad US/metric weight setting (or the MCDU Units button, Alt+U, inside the MFD window) switches MSFSBA's weight readouts — gross weight and total fuel — between kilograms and pounds, and is announced when it changes. The standby ISIS altitude always stays in feet (it is the backup instrument).

State changes

Engine state (Off/On/Starting/Shutting Down), engine masters and crossfeeds, doors, wheel chocks, safety cones, external power available, ATC message waiting, destination fuel OK/below-minimum, and the E/WD memo callouts (PARK BRK ON, APU AVAIL, ELEC EXT PWR, and so on).

10. Settable controls and the systems panels

Browse the systems with Ctrl+1 (Sections) and Ctrl+2 (Panels). Almost every cockpit control is a combo box you set by picking a value; numeric controls are edit fields. The highlights:

Altimeter / baro

Captain and First Officer baro: STD/QNH push-pull combos, a "Set QNH" numeric field (in your current unit; it drives both altimeters), and the unit readout. The standby (ISIS) baro is also settable.

EFIS (per side)

LS, V/V, Constraints, Airport, Traffic on/off; ND mode (Rose ILS/VOR/Nav, Arc, Plan); ND range; navaid 1 & 2 (Off/ADF/VOR); ND filter (Waypoints/VOR-DME/NDB); overlay (Off/Weather/Terrain); OANS range.

Pressurization / air conditioning

Cabin altitude mode and V/S mode (Off/Auto), ditching, the manual cabin-altitude and V/S target knobs (numeric, in manual mode), pack/hot-air/ram-air/crossbleed, pack flow, cabin fans, air extract, cargo isolation and heater, and cockpit/cabin/cargo target temperatures.

ECAM control panel

ECAM ALL, Abnormal Proc, Checklist, Clear, Recall, T.O Config, Emergency Cancel, Up/Down/More, plus the SD page selector.

Transponder / radios / audio

Squawk code (edit field), transponder mode and ident, ATC altitude reporting. The RMP panel shows the three RMP states plus the VHF 1–3 active and standby frequencies as a quick read-only glance.

To tune the radios, open the dedicated Radio Management Panel window with Ctrl+Shift+R in input mode. It works like the accessible MCDU: the RMP screen is read-only text you read with the arrow keys (VHF 1/2/3 active and standby, which is transmitting and selected, the message line), it refreshes automatically, and it announces the standby and any message (such as "VHF FREQ NOT VALID") as they change. There is a Captain / First Officer selector, and everything else is a keyboard shortcut so nothing clutters the window. The soft keys use the same scheme as the MCDU — left keys on Ctrl, right keys on Alt: Ctrl+1/2/3 select radio 1/2/3 (press again to load), Alt+1/2/3 swap that radio's active and standby. The A380 RMP has two pages, reached with the VHF page and Transponder page buttons (in the Tab order) or Alt+V / Alt+Q. (HF, TEL, NAV and MENU are not modelled by FlyByWire on the current build, so those keys do nothing.)

You set frequencies by typing the digits right on the screen — there is no separate entry box. With the screen focused (where you start), select the radio, then just type the frequency: each digit is keyed into the RMP as you type it (the screen is read-only, so the digits go to the radio, not the text), and the standby auto-completes live (type 8 for 118.000, 1 1 8 5 0 for 118.500; no decimal point, Backspace deletes a digit). On the VHF page: Enter loads what you typed into standby, then Alt+1/2/3 swaps it active (the pilot's call). Alt+C fully clears a stuck entry. On the Transponder (SQWK) page the screen reads the mode (AUTO or STBY), the squawk code and IDENT; type the four squawk digits, press Enter to validate them, and Alt+3 sends IDENT. Everything updates when you switch Captain ↔ First Officer — the screen, the announcements and the keys all follow the selected RMP. The old stock-radio panel was removed — on the A380 all tuning goes through the RMP. The Audio Control Panel (Pedestal) still exposes the captain's RMP-1 receive selectors — VHF 1, VHF 2, cabin interphone, interphone, PA and NAV — so you can confirm which sources you are listening to.

Fuel system

The feed-tank pumps are settable: Feed Tank 1–4 main and standby pumps (each on/off). MSFSBA toggles them through the aircraft's electrical-circuit model, so a set takes effect immediately and the true state is read back. Also here: fuel transfer and jettison, crossfeed valves, per-engine fuel used, and the total/per-tank fuel quantities (which honour the kg/lb unit toggle).

Flight controls and trim

Pitch trim (the THS) reads in degrees up or down, matching the System Display. Rudder trim reads as left/right degrees with a Rudder Trim Reset control, and there is a nosewheel-steering pedal disconnect (held during a rudder-pedal flight-control check). Surface deflections (elevator, aileron, rudder, spoilers) and the sidestick priority-takeover indications are readable. The continuous trim-announce toggle is Shift+T in OUTPUT mode.

Clock and chronometer

The cockpit clock is fully readable in the Clock panel. The Chronometer (CHR stopwatch) reads as minutes and seconds; start/stop it with the Chronometer Start / Stop control and zero it with Chronometer Reset. The Elapsed Time counter reads as hours and minutes and is driven by the Elapsed Time knob (Run / Stop / Reset). A counter that has been reset reads "Reset".

Surveillance (SURV) and metric altitude

The MFD SURV pages (reach them with Alt+A's neighbouring SURV page, or the page combo) expose the transponder/TCAS mode and range, the weather-radar and predictive-windshear/turbulence controls, and TAWS terrain/GPWS modes. The FCU metric-altitude (MTRS) button is a settable control here too; its effect on every altitude readout is described in Part 9.

Autoflight, gear, brakes, engines, lights, fuel, electrics, hydraulics

Autopilot 1/2, localizer, approach, expedite; autobrake (Disarm/BTV/Low/L2/L3/High) and RTO arm; gear and gravity extension; flaps, parking brake, speedbrake, thrust-lever detents; engine masters/mode/start; all exterior and interior lights; fuel transfer/jettison/crossfeed; ELEC, HYD, BLEED, ADIRS, the PRIM/SEC flight-control computers; GPWS/TAWS; weather radar; ground services (doors, jet bridge, stairs); and "Load Aircraft Preset" (Cold & Dark / Powered / Pushback / Taxi / Takeoff).

A few overhead pushbuttons (Flight Director, some packs/bleeds/bus-ties and the like) are computed outputs on this build: the FBW systems recompute them every frame, so an external "set" reverts. MSFSBA still reads their true state and announces changes; they are simply not settable from outside until FBW provides a control. This is a platform limitation, not a missing feature.

11. OANS airport map and Brake-To-Vacate (BTV)

The A380's OANS (Onboard Airport Navigation System) shows a moving airport map on the navigation display, and BTV (Brake-To-Vacate) lets the autobrake stop the aircraft exactly at a chosen runway exit. Open the accessible OANS/BTV window with Ctrl+Shift+B.

The OANS control panel only draws on the navigation display at the closest "airport" ND range, so opening the window automatically zooms the Captain ND to the airport map for you (you do not see the ND, so this is harmless) — give it a second after opening, and press F5 if it still says it is loading. Read the controls with the arrow keys; Enter activates a button or opens a list, you type a code into a field and press Enter, and Escape closes. Across the top are three tabs — arrow to one and press Enter:

  • ARPT SEL — pick the airport: choose the entry mode (ICAO / IATA / CITY NAME), type the code into the field and press Enter, then activate DISPLAY AIRPORT. ORIGIN / DEST / ALTN jump straight to your flight-plan airports.
  • MAP DATA — Brake-To-Vacate: pick the landing runway, read RUNWAY LENGTH and the BTV STOP DISTANCE, and choose the exit (a taxiway, a stand, or OTHER). BTV MANUAL CONTROL is the fallback. You can also set a landing shift here.
  • STATUS — the active / second airport database and a SWAP, plus the database identifier.

To arm BTV, set the autobrake selector to BTV (confirm on the FMA), choose the landing runway, then a runway exit beyond the "wet" stopping line. After touchdown BTV optimises braking to reach that exit at about 10 knots.

The OANS map needs airport data: it works best with a Navigraph subscription, and only populates once the inertial system (ADIRS) is aligned and you are at or near an airport (on the ground or on approach). Away from an airport it will say it is not available — that is correct, it is a ground / terminal-area tool. For taxi guidance with turn-by-turn directions, MSFSBA's separate Taxi Assist feature (the Y / Shift+Y keys) is the primary tool — it uses the navigation database directly.

Rollout callouts. When BTV is doing the braking, MSFSBA speaks the BTV state ("Brake To Vacate, Decel", "End of Braking") and counts down the distance to your selected exit and the runway remaining as they shrink (for example "500 meters to exit", "300 meters runway remaining"), at a few thresholds so it is informative without being chatty. These only speak during the actual braking rollout. Mute any of them from the Monitor Manager (Ctrl+M): "BTV Distance to Exit", "Runway Remaining", "Brake To Vacate".

Independently of BTV, the ROW/ROP runway-overrun protection is active whenever an autobrake mode is armed, and MSFSBA speaks its callouts on approach and rollout.

12. A walkthrough: a complete flight

This ties the pieces together. Adjust to your route and procedures.

On the ground (cold and dark → ready)

  1. Load the A380 and let MSFSBA detect it. If starting cold and dark, either run the checklist or load a Powered/Ready preset (Panels → Load Aircraft Preset, or the flyPad Presets app).
  2. flyPad (INPUT Shift+T): on Dispatch, set your payload and fuel; in Settings, set ADIRS alignment time and your SimBrief ID.
  3. MCDU (INPUT Shift+M): INIT → load the SimBrief route (5a). Then DEPARTURE (5b), FUEL & LOAD and PERF (5c), and ARRIVAL (5d). Clear any discontinuity.
  4. Use the live checklist (Part 7) to work the cockpit-prep and before-start items; sensed items tick as you do them.

Taxi and takeoff

  1. Request pushback and engine start via the flyPad Ground app or the overhead.
  2. For taxi, open Taxi Assist (INPUT Shift+Y), enter your ATC taxi clearance, and follow the turn-by-turn guidance (Y reads status, Ctrl+Y repeats).
  3. Set takeoff config; the FMA, thrust-lever detents and flap settings all announce. On the takeoff roll, autothrust and SRS modes call out.

Climb, cruise, descent

  1. The FMA announces every mode change. Check progress any time with D (distance to destination) and, once the FMS has computed the descent, Shift+D (distance to top of descent).
  2. Use the FCU set/read keys (Part 13) to manage heading, speed, altitude and vertical speed; each push/pull confirms managed vs selected.
  3. Monitor systems via the SD pages and the auto-announced cautions. "Top of descent reached" announces as a PFD message.

Approach and landing

  1. Confirm the approach is set and the minimums are entered (PERF APPR). Arm the approach; the FMA announces capability (CAT 1/2/3).
  2. If using BTV, arm it and pick your exit (Part 11). Otherwise an autobrake mode still gives you ROW/ROP overrun protection.
  3. On final and rollout, listen for the overrun callouts and use the Landing Exit Planner (INPUT Shift+X) if you want exit guidance.

14. FlyByWire A380X feature status and limitations

The A380X (development build) is under active development. Most of it works well, but a few things are incomplete on the FlyByWire side — not on MSFSBA's — and because it is the rolling dev build, the exact status changes over time. Knowing what is and isn't finished saves confusion:

  • Modeled: the FMS (with direct-to on airways, FIX INFO, step climbs), autothrust and full fly-by-wire, the EWD/SD ECAM with hundreds of procedures and memos, the 16 SD pages, Green/Yellow 5000-psi hydraulics, electrics/pneumatics/pressurization/APU/fire, Trent 972 engines, RMP, ISIS, OANS+BTV+ROW/ROP, and the flyPad.
  • Experimental / in progress: a native MSFS 2024 build (current 2024 support is a compatibility layer), the Vertical Display, and the OIT/OIS onboard terminal (only a flight log and company-message inbox so far).
  • Incomplete / disabled: FLS / F-G/S approaches (non-functional), the second ISIS (disabled), and OANS airport data that depends on a Navigraph subscription.
  • Status/INOP/limitations lists: the data MSFSBA needs to read the STATUS page's inoperative-systems and limitations lists is reachable, but the FBW development build does not fully populate it yet; MSFSBA will surface it as soon as FBW ships the data.
  • D / Shift+D distance hotkeys: distance to destination works on the ground and in flight; distance to top of descent only reads once the FMS has computed a descent (in flight). ETA and landing-fuel are planned additions once the in-flight prediction data is verified.
  • Oxygen pressure (PSI) is not simulated. The FBW build shows a fixed crew/passenger oxygen pressure on the DOOR/OXY page — there is no underlying variable, so MSFSBA cannot read a live value. The oxygen status that is modelled (passenger masks deployed, passenger-oxygen light, the timer-reset control and its fault) is read and announced.
  • Chronometer: the clock CHR stopwatch and ET elapsed-time counter are modelled and read live (formatted as minutes/seconds and hours/minutes) — they run only when the clock is powered (DC essential bus).
  • Fluid quantities (hydraulic reservoirs in gallons, engine oil in quarts, fuel in kg) are read live and were verified against the FBW source; engine oil pressure is not exposed because the build's value is unreliable.

Because it is a development build, occasional control or readout names may differ slightly from what is documented here; report anything that reads wrong and it can be corrected.

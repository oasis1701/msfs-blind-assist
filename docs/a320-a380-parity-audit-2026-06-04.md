## Headline finding

**The A320 is far closer to parity than any prior doc or the raw agent sweep suggested.** The two
def files use **different internal idioms** (the A320 does NOT use the A380's `p[...]`/`d[...]` dict +
`Mon`/`OffAuto` helpers), so an A380-pattern diff manufactures false gaps. After reconciling against
the A320's real `GetPanelDisplayVariables`, `GetHotkeyVariableMap`, `SdSystemRows`, and live reads,
**~80% of the agent-reported "gaps" are false positives** (already done, or A320-by-design). The
genuine actionable list is short.

## Methodology тАФ reliable gap-finding (do NOT trust agent diffs)

Dispatching agents to "diff the two aircraft" over-reports ~80-90%. Root causes seen this session:
1. **Wrong identifier** тАФ the cockpit XML names controls by 3D NODE_ID (`PUSH_OVHD_ELEC_BAT1`); MSFSBA
   uses the L:var (`A32NX_OVHD_ELEC_BAT_1_PB_IS_AUTO`). A node-id diff marks everything missing (one
   agent reported 181 false gaps incl. batteries the A320 obviously has). Match by L:VAR token only.
2. **Idiom assumption** тАФ the A320 def uses explicit `SimVarDefinition` objects (not the A380's
   `p[]`/`d[]`/`Sel`/`Mon`), and a runtime loop (`FlyByWireA320Definition.cs` ~line 4734) flips
   discrete combos `OnRequest`тЖТ`Continuous+IsAnnounced` at run time, so the raw `OnRequest` text is not
   the effective behaviour.
3. **Alternate surfaces** тАФ many controls are exposed via dedicated WINDOWS (Autopilot Ctrl+P, Baro
   Ctrl+B, FCU windows, MCDU), invisible to an agent reading only panel arrays.
4. **0-read ambiguity** тАФ an UNDEFINED L:var reads 0 via `GetSimVarValue`, same as a defined-off one;
   live probes alone can't prove non-existence тАФ only the FBW source can.
5. **Recall bias** тАФ agents pad with confident low-confidence rows; a parity audit needs precision.

**Reliable protocol тАФ verify EVERY candidate 3 ways before implementing:** (1) FBW source
(`D:\Claude\fbw\aircraft` тАФ `a320-simvars.md` for var+value mapping, `A320_NEO_INTERIOR.xml` inline
`<SIMVAR_NAME>`/`<TOGGLE_SIMVAR>`/`<HOLD_SIMVAR>` + ANIMTIP for label/momentary-vs-latching); (2) grep
the BARE token across ALL of `MSFSBlindAssist`; (3) live write-stick via calc path, read back in a
SEPARATE eval. Use agents only for mechanical EXTRACTION ("list every inline SIMVAR binding in this
file"), never for the verdict.

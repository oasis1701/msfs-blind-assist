# PMDG 777 Phase 2: Bug Fixes, Missing Systems & Monitoring Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix all functional bugs and misconfigurations in the PMDG 777 definition, add missing flight-critical systems (engine fire handles, doors, FMC data, trim), and enhance background monitoring.

**Architecture:** All changes are in `PMDG777Definition.cs` (variable definitions, event maps, panel controls, ProcessSimVarUpdate, HandleHotkeyAction) and `HotkeyManager.cs` (new HotkeyAction enum entries). The PMDG777XDataStruct already contains all needed fields — no struct changes required.

**Tech Stack:** C# 13 / .NET 9, Windows Forms, PMDG SDK via SimConnect Client Data Areas

case HotkeyAction.ReadDistanceToTOD:
{
    var dm = simConnect.PMDG777DataManager;
    if (dm == null) return false;
    float dist = (float)dm.GetFieldValue("FMC_DistanceToTOD");
    if (dist < 0)
        announcer.AnnounceImmediate("Top of descent not available");
    else if (dist < 0.1f)
        announcer.AnnounceImmediate("Past top of descent");
    else
        announcer.AnnounceImmediate($"{dist:F0} miles to top of descent");
    return true;
}

case HotkeyAction.ReadDistanceToDest:
{
    var dm = simConnect.PMDG777DataManager;
    if (dm == null) return false;
    float dist = (float)dm.GetFieldValue("FMC_DistanceToDest");
    if (dist < 0)
        announcer.AnnounceImmediate("Distance to destination not available");
    else
        announcer.AnnounceImmediate($"{dist:F0} miles to destination");
    return true;
}

case HotkeyAction.ReadThrustLimitMode:
{
    var dm = simConnect.PMDG777DataManager;
    if (dm == null) return false;
    int mode = (int)dm.GetFieldValue("FMC_ThrustLimitMode");
    string modeText = mode switch
    {
        0 => "None",
        1 => "TO", 2 => "TO 1", 3 => "TO 2",
        4 => "D-TO", 5 => "D-TO 1", 6 => "D-TO 2",
        7 => "CLB", 8 => "CLB 1", 9 => "CLB 2",
        10 => "CRZ", 11 => "CON",
        _ => $"Unknown ({mode})"
    };
    announcer.AnnounceImmediate($"Thrust limit {modeText}");
    return true;
}

## Post-Implementation Notes

### Door array index mapping
The DOOR_state[0-15] → physical door mapping (Entry 1L, 1R, 2L, etc.) is based on the SDK event order (EVT_DOOR_1L through EVT_DOOR_EE_ACCESS at offsets 14011-14026). This may need verification in-sim — if door labels don't match reality, update the `Name` field array indices accordingly.

### Engine fire handle interaction sequence
The engine fire handle unlock → pull → discharge sequence may need fine-tuning in-sim. The `_guardedMap` entry for unlock buttons sends unlock → fire handle TOP in sequence, but the multi-position handle (Normal → Pulled → Disch Left → Disch Right) uses wheel-step logic. If the handle doesn't respond correctly to wheel steps, custom HandleUIVariableSet logic may be needed.

### Thrust limit mode values
The FMC_ThrustLimitMode mapping (0-11) is based on the SDK documentation. The actual values may extend beyond 11 depending on aircraft variant. The `_ => $"Unknown ({mode})"` fallback handles unexpected values gracefully.

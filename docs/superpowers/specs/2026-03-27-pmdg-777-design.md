# PMDG Boeing 777 Accessibility Implementation Design

## Overview

Add full PMDG Boeing 777 support to MSFS Blind Assist, making all cockpit systems accessible to blind sim pilots via screen reader. This includes all overhead, forward panel, glareshield, and pedestal controls, plus full CDU access.

The PMDG 777 uses a fundamentally different data architecture from Fenix/FlyByWire aircraft: binary Client Data Areas instead of individual LVars. This requires a new data management layer while keeping the existing UI and announcement pipeline unchanged.

## Key Design Decisions

1. **CDU interaction**: Mirror Fenix MCDU approach (ListBox for display rows, TextBox for scratchpad, buttons for function/page keys)
2. **Control types**: ComboBoxes for all switches and multi-position selectors (including ON/OFF toggles)
3. **Momentary pushbuttons**: Button controls with automatic annunciator state readback after press
4. **Panel granularity**: Fine-grained panels matching physical cockpit groupings
5. **Data integration**: Native C# PMDG SDK Client Data Areas (no MCP server dependency)
6. **MCP value setting**: Direct-set events exclusively (`EVT_MCP_*_SET`), no dial increment/decrement
7. **Event dispatch**: ROTOR_BRAKE via MobiFlight WASM for standard events, SetClientData for MCP direct-set events
8. **No mouse flags needed**: Simple ROTOR_BRAKE offset calculation works for all standard toggle events
9. **Guarded switches**: Automatic open guard -> toggle -> close guard sequence, transparent to user

## Architecture

### New Components

#### 1. PMDG777DataManager (SimConnect/PMDG777DataManager.cs)

Core data layer bridging PMDG's binary Client Data Areas with the app's variable system.

**Responsibilities:**
- Register PMDG Client Data Areas via SimConnect: `PMDG_777X_Data` (684 bytes, read), `PMDG_777X_Control` (8 bytes, write), `PMDG_777X_CDU_0/1/2` (CDU screens)
- Poll and deserialize the 684-byte `PMDG_777X_Data` struct at ~1 second intervals
- Detect field-by-field changes against previous snapshot, raise `SimVarUpdated` events for changes
- Send control events via two mechanisms:
  - Standard events (offset < 14500): RPN calculator code `"{offset*100+1} (>K:ROTOR_BRAKE)"` via MobiFlight WASM
  - Direct-set events (offset >= 14500): 8-byte struct `{event_id, parameter}` written to `PMDG_777X_Control` Client Data Area
- Read and parse CDU screen data (24 columns x 14 rows, per-cell color/flags)
- Provide `SendGuardedToggle(guardEvent, switchEvent)` helper that sequences: open guard -> delay -> toggle -> delay -> close guard

**CDU reading:**
- Parsed into 14 rows of 24 characters as plain text strings
- Exposed via `GetCDUScreen(int cdu)` returning row strings
- Polled on demand when CDU form is open (500ms interval), not continuously

#### 2. PMDG777XDataStruct (SimConnect/PMDG777XDataStruct.cs)

C# struct with `[StructLayout(LayoutKind.Sequential)]` mirroring the 684-byte PMDG binary data structure. Uses default MSVC alignment (no `Pack=1`). Fields include:
- Boolean switches (`bool`)
- Unsigned byte selectors/knobs (`byte`)
- Unsigned short values like heading, altitude (`ushort`)
- Signed short values like vertical speed (`short`)
- Float values like IAS/Mach, fuel quantities (`float`)
- Byte arrays for indexed fields (e.g., `bool[2]` for left/right engines)

#### 3. PMDG777Definition (Aircraft/PMDG777Definition.cs)

Aircraft definition implementing `IAircraftDefinition` via `BaseAircraftDefinition`.

**Core properties:**
- `AircraftName`: "PMDG 777"
- `AircraftCode`: "PMDG_777"
- FCU control types: `SetValue` for all (heading, speed, altitude, VS)

**Overrides:**
- `GetVariables()`: All PMDG variables using `SimVarType.PMDGVar` type
- `GetPanelStructure()`: Section -> panel hierarchy
- `BuildPanelControls()`: Panel -> variable key mappings
- `HandleUIVariableSet()`: PMDG event dispatch for control changes, including guarded switch sequences
- `ProcessSimVarUpdate()`: Custom MCP readout announcements, annunciator state formatting
- `HandleHotkeyAction()`: CDU form, MCP direct-set dialogs, fuel/weight readouts, PMDG-specific readouts

#### 4. PMDG777CDUForm (Forms/PMDG777/PMDG777CDUForm.cs)

CDU dialog mirroring Fenix MCDU pattern.

**Layout:**
- Connection status label ("CDU Connected" / "CDU Not Powered")
- Display ListBox: 14 rows, monospace, read-only. Screen reader reads each line when arrowed to.
- Scratchpad TextBox: Type text + Enter sends sequential CDU key presses. Backspace sends CLR.
- Line Select buttons: L1-L6 and R1-R6
- Page function buttons: INIT REF, RTE, DEP ARR, ALTN, VNAV, FIX, LEGS, HOLD, FMC COMM, PROG, MENU, NAV RAD, PREV PAGE, NEXT PAGE
- Special buttons: EXEC, CLR, DEL

**Keyboard shortcuts within CDU form:**
- F1-F6 for L1-L6, Shift+F1-F6 for R1-R6
- Page Up / Page Down for PREV PAGE / NEXT PAGE
- Ctrl+Enter for EXEC

**Screen reading behavior:**
- Title row (row 0) change announced: "Page: RTE 1"
- Scratchpad changes announced automatically
- Line select key press announces the row content at that position
- "Read current page" function announces title + all non-empty rows

**CDU event mapping:**
All CDU keys sent via ROTOR_BRAKE: `EVT_CDU_L_L1` through `EVT_CDU_L_R6`, `EVT_CDU_L_INIT_REF`, `EVT_CDU_L_A` through `EVT_CDU_L_Z`, `EVT_CDU_L_0` through `EVT_CDU_L_9`, `EVT_CDU_L_EXEC`, `EVT_CDU_L_CLR`, `EVT_CDU_L_DEL`, `EVT_CDU_L_SLASH`, `EVT_CDU_L_SPACE`, etc.

**Polling:** 500ms when form is open. CDU 0 (captain) default, selector to switch to CDU 1 (center) or CDU 2 (FO).

### SimConnectManager Changes (Minimal)

1. New `SimVarType.PMDGVar` enum value
2. `PMDG777DataManager` property, instantiated when PMDG 777 selected, disposed when switching away
3. `OnRecvClientData` handler routes PMDG CDA responses to data manager
4. `SendPMDGEvent(string eventName, int? parameter)` public method delegating to data manager
5. Registration bypass: `PMDGVar` type variables skip normal `AddToDataDefinition`, data manager handles its own subscriptions

### MainForm Changes (Minimal)

1. "PMDG 777" menu item in `MainForm.Designer.cs` + click handler
2. `LoadAircraftFromCode()` switch: `"PMDG_777"` case instantiating `PMDG777Definition`
3. `HandleUIVariableSet` routing: `PMDGVar` type calls `SendPMDGEvent()` instead of `SetLVar()`
4. Panel variable requests: for PMDG, request fresh data snapshot from data manager
5. CDU hotkey: `ShowFenixMCDU` action opens `PMDG777CDUForm` when current aircraft is PMDG 777

### No Changes Needed To

- HotkeyManager (aircraft-agnostic)
- ScreenReaderAnnouncer (receives strings from any source)
- UI generation / PanelControlFactory (driven by SimVarDefinition properties)
- BaseAircraftDefinition (PMDG777Definition inherits and overrides)
- Any Fenix-specific code (fully separate)

### Continuous Monitoring (Announced Automatically)

- All `_annun` variables: announced on state change as "[System] [description] light [on/off]"
- MCP annunciators: AP L/R, AT, LNAV, VNAV, FLCH, HDG HOLD, VS/FPA, ALT HOLD, LOC, APP
- Fuel quantities: center, left, right, aux (on request via hotkey)
- FMC data: V-speeds, thrust limit mode, distances (on request via hotkey)
- Door states (16 doors)
- Only announce transitions, not steady states (prevents spam during panel loading)

## Control Interaction Patterns

### Boolean Switches (_Sw_ON, _Sw_OFF) -> ComboBox

- ComboBox with "Off" / "On"
- Selection change sends toggle event via ROTOR_BRAKE (no mouse flags needed)
- State confirmed by reading back from data broadcast

### Multi-Position Selectors (_Selector) -> ComboBox

- ComboBox with named positions (e.g., Off/Auto/On, RTO/Off/Disarm/1/2/3/Max Auto)
- Selection change sends WHEEL_UP or WHEEL_DOWN mouse flags to step to desired position
- For selectors supporting direct position values, send position directly

### Momentary Pushbuttons (_Sw_Pushed) -> Button

- Button labeled with function name (e.g., "LNAV")
- Press sends event via ROTOR_BRAKE
- After press, reads associated annunciator and announces state: "LNAV engaged" / "LNAV disengaged"
- When tabbing to button, current annunciator state included: "LNAV (engaged)"

### MCP Value Displays -> Read-Only Labels + Hotkey Dialogs

- `MCP_IASMach`, `MCP_Heading`, `MCP_Altitude`, `MCP_VertSpeed` shown as labels in MCP panel
- Values set via hotkey input dialogs using direct-set events
- Encoding: IAS = raw knots, Mach = value * 1000, Heading = raw degrees, Altitude = raw feet, VS = value + 10000

### Continuous Knobs (_Knob, 0-100) -> ComboBox

- Discretized to useful positions: Off (0), Dim (25), Medium (50), Bright (75), Full (100)
- Specific knobs may use different labels appropriate to their function (e.g., temperature knobs: Cold/Cool/Medium/Warm/Hot)
- Not exposing full 0-100 range for accessibility

### Guarded Switches -> Transparent Automatic Sequence

- When user changes a guarded switch via ComboBox, `HandleUIVariableSet()` sends:
  1. Open guard event
  2. Brief delay (~100ms)
  3. Toggle switch event
  4. Brief delay (~100ms)
  5. Close guard event
- User only sees the result announcement (e.g., "EEC Mode Left: Alternate")
- No separate guard interaction or verbose guard announcements

### Annunciator Monitoring

- All `_annun` variables monitored continuously
- On state change: announce "[System] [Description] light [on/off]"
- Example: `HYD_annunPrimaryElecPumpFAULT_1` goes true -> "Hydraulic primary electric pump 1 FAULT light on"
- Only announce transitions, not steady states

## Data Flow

### Simulator -> UI Update Path

```
PMDG 777 sim -> PMDG_777X_Data Client Data Area (684 bytes)
  -> PMDG777DataManager deserializes struct
  -> Compares field-by-field against previous snapshot
  -> For each changed field: raises SimVarUpdated event
  -> MainForm.OnSimVarUpdated() queues event
  -> ProcessEventBatch() (33ms timer)
  -> aircraft.ProcessSimVarUpdate() for custom handling (MCP readouts, annunciator formatting)
  -> If not custom-handled: generic AnnounceVariableState()
  -> UpdateControlFromSimVar() syncs UI if panel is open
```

### UI -> Simulator Command Path

```
User changes ComboBox / clicks Button
  -> Control event handler
  -> aircraft.HandleUIVariableSet() (PMDG-specific logic)
  -> For standard toggles: PMDG777DataManager generates RPN "{offset*100+1} (>K:ROTOR_BRAKE)"
  -> For guarded switches: open guard -> delay -> toggle -> delay -> close guard
  -> For MCP direct-set: PMDG777DataManager writes 8-byte struct to PMDG_777X_Control CDA
  -> Data broadcast update confirms new state
  -> Screen reader announces result
```

### CDU Data Flow

```
CDU form opens -> starts 500ms polling timer
  -> PMDG777DataManager.GetCDUScreen(0) reads PMDG_777X_CDU_0 CDA
  -> Parses 24x14 character grid into 14 row strings
  -> Compares with previous screen
  -> Updates ListBox rows, announces title/scratchpad changes
CDU form closes -> stops polling
```

## Technical Notes

### PMDG Event Dispatch

Two methods, automatically selected by event offset:

- **Offset < 14500 (standard cockpit events):** RPN calculator code `"{offset*100+1} (>K:ROTOR_BRAKE)"` sent via MobiFlight WASM module. No mouse flag parameters needed for simple toggles.
- **Offset >= 14500 (MCP direct-set events):** 8-byte binary write `{uint32 event_id, uint32 parameter}` to `PMDG_777X_Control` Client Data Area via `SetClientData()`.

### PMDG Data Structure Alignment

The `PMDG_777X_Data` struct uses default MSVC alignment (NOT packed). The C# struct must match this with `[StructLayout(LayoutKind.Sequential)]` without `Pack=1`. Total size: 684 bytes.

CDU structs use byte alignment (`Pack=1`).

### MCP Value Encoding

| Parameter | Encoding | Example |
|-----------|----------|---------|
| IAS | Raw knots | 250 |
| Mach | Value * 1000 | 0.78 -> 780 |
| Heading | Raw degrees (0-359) | 250 |
| Altitude | Raw feet | 35000 |
| Vertical Speed | Value + 10000 | -1800 fpm -> 8200, +2000 fpm -> 12000 |
| FPA | Value * 10 + 100 | -1.8 -> 82 |

### Verified Behaviors (Tested Live)

- Non-guarded toggle switches work with simple ROTOR_BRAKE events (no parameters): Battery, APU Gen confirmed
- Guarded switches require guard open -> toggle -> guard close sequence: EEC Mode confirmed
- Guard events are simple toggles (send once to open, again to close)
- Mouse flag parameters (MOUSE_FLAG_LEFTSINGLE etc.) are NOT needed for ROTOR_BRAKE events
- PMDG data broadcast provides real-time state for all cockpit fields
- CDU screen data available when aircraft is powered (returns null when unpowered)

FLYBYWIRE A380X HOTKEY REFERENCE

This is the FlyByWire A380X (development build). The MCDU is actually the MFD
(driven by the KCCU cursor, not classic line-select keys); the EFB is the
flyPad. Both are read and driven live through the MSFS Coherent GT debugger —
no add-on install and no Developer Mode are required; the A380X just needs to
be loaded with the cockpit displays powered up.

Panels: use the main window's Sections and Panels lists (Ctrl+1 / Ctrl+2) to
browse every system. Most controls are combo boxes or buttons; many panels also
have a read-only "Status Display" field listing the live readouts one per line
(battery voltages, gen load, hydraulic pressures, bleed temps, brake temps,
engine N2/N3/EGT/FF, fault lights, etc.). Press the panel's Refresh button (or
re-open the panel) to update on demand; values also update automatically as the
sim changes.

Application Navigation:
  Ctrl+1     Focus Sections List
  Ctrl+2     Focus Panels List

Tip: Press the same bracket key again to cancel a mode you activated by accident.


Displays and Windows (direct chords — no bracket mode needed):
  Ctrl+M     A380 Monitor Manager — tick/untick which variables are announced
             automatically as they change (master warning/caution, engine state,
             fire, flight phase, ATC message waiting, the annunciator/fault lights
             below, and a single "ECAM E/WD call-outs" entry that mutes the live
             memo/warning read-out). Same Ctrl+M chord the Fenix and PMDG use;
             your choices are saved between sessions.

  Ctrl+E     Toggle ECAM E/WD monitoring on/off — the same as un/re-ticking the
             "ECAM E/WD call-outs" entry in the Monitor Manager, but as a quick
             chord. Announces "E W D monitoring enabled/disabled". Persisted.

  Ctrl+Shift+B  A380 Airport Map and BTV (OANS) window — the Onboard Airport
             Navigation System control panel from the ND, shown as an accessible
             web page (same browser view as the flyPad). Use it for BTV
             (Brake-To-Vacate): select the landing runway and the exit/taxiway to
             vacate at, search airports/runways/taxiways/stands, LDG SHIFT, and
             read RUNWAY LENGTH / LDA / BTV STOP DISTANCE. To ARM BTV itself, set
             the Autobrake selector (Pedestal) to BTV. NOTE: the airport map
             controls only populate once the OANS has an aircraft position — the
             ADIRS must be aligned (NAV) and you must be at or near an airport;
             until then the window says "OANS not available yet".

Annunciators and fault lights are announced automatically as they change
  (master warning/caution, APU EGT warning, thrust-lever warning, performance
  warning, TCAS fault; overhead pushbutton FAULT lights for batteries, gens,
  IDGs, engine + electric hydraulic pumps, bleeds, packs, hot air, ram air,
  cargo isolation/heater, TADD/FDAC/OCSM/VCM channels, fire squibs, etc.). If
  any of these are too chatty, un-tick them in the Monitor Manager (Ctrl+M).

Read-out windows (open from OUTPUT mode — see the PFD/ND/ECAM keys below):
  - PFD window: flight-mode annunciator (lateral/vertical mode), approach
    capability (CAT 1/2/3), and PFD messages. FMA mode changes (OP CLB, V/S,
    FPA, G/S, LAND, FLARE, SRS, NAV, LOC, RWY...) are also announced live.
  - Navigation Display window: ND mode and range, the TO waypoint (ident,
    distance, bearing, ETA), cross-track error, RNP, and ILS deviation.
  - Standby Instruments window (ISIS) — Alt+I: standby attitude (pitch/bank),
    heading, indicated airspeed + Mach, standby altitude with its baro reference
    (STD/QNH) and setting, and the LS overlay state. Duplicates the universal
    altitude/airspeed keys, gathered as one standby cross-check panel.
  - System Display window (ECAM/STATUS): a LIVE read of the real System Display.
    Pick any SD page from the "SD page" list (Engine, APU, Bleed, Air Cond,
    Pressurization, Doors, Electrical AC/DC, Fuel, Wheel, Hydraulics, Flight
    Controls, Circuit Breakers, Cruise, Status, Video) and the window commands the
    real System Display to that page AND shows exactly what it displays, decoded
    into readable lines (oxygen pressures, per-tank fuel, gross weight, CG,
    temperatures, pressures, door states, ...). Refresh button + F5. (If the live
    read can't reach the display it falls back to the SimVar decode. The Cruise and
    Status pages are still being filled in by FlyByWire, so they may read mostly
    placeholders for now.)
  - E/WD window — Alt+E: a live read of the Engine/Warning Display (per-engine
    throttle/N1/EGT/fuel-flow, the memo lines, and any active warnings and ECAM
    action items). Alt+E also speaks the current warnings immediately.
  Each window has a Refresh button and F5; Escape closes. The PFD, ND and ISIS
  windows also have a "Live scrape" button (or F6) that opens a live read of the
  real display — useful for text the SimVar readout doesn't cover (note the PFD/
  ND/ISIS speed/altitude/attitude tapes read as scale marks, so the normal SimVar
  readout stays the primary source for those).

Thrust levers: moving the levers announces the detent reached (Idle / Climb /
  Flex MCT / TOGA / Reverse). You can also set a detent from the Thrust Levers
  panel (All Thrust Levers, or per engine) in the Pedestal section.

Checklist (Output mode then the checklist key) opens the LIVE Electronic
  Checklist (ECL) window — it reads the aircraft's own normal checklists and any
  active ECAM abnormal procedure straight from the E/WD, so it always matches the
  real cockpit, and it is FULLY interactive.
  How to use it:
    1. Press "Show / hide (C/L)" (Alt+S) to bring the checklist up.
    2. Arrow Up/Down through the list — this moves the real checklist cursor and
       reads each line, telling you whether an item is checked, and whether it is
       a manual or sensed action.
    3. Press Enter (or the "Check / select" button, Alt+C):
         - on the checklist menu, this OPENS the highlighted checklist;
         - inside a checklist, this TICKS the highlighted manual item and moves on.
    4. "Clear / back" (Alt+L) returns to the menu; "Abnormal" (Alt+A) opens the
       abnormal-procedure list.
  Items also tick themselves the moment the aircraft senses the action (set the
  park brake and PARK BRK ON ticks itself, announced as "PARK BRK ON, checked").
  Abnormal procedures present automatically when a failure occurs. F5 refreshes,
  Escape closes.

  Which way do I access checklists? Use the Checklist hotkey above — that window
  IS the workflow. You do NOT go through a combo box. The ECAM Control Panel does
  have a "Checklist" entry (and Up / Down / Abnormal Proc / Clear), but those are
  just the physical ECP push-buttons: the "Checklist" one only toggles the
  checklist ON the E/WD screen (same as the window's Show / hide). It does not let
  you read or tick anything. There is nothing to "set" in that combo and then Tab
  to — everything you need (reading lines, opening a checklist, ticking items) is
  in the Checklist window, driven by the arrow keys and Enter as described above.

  Note on the flyPad's own Checklists page (the tablet has one too): that page is
  a SEPARATE, tablet-only tracker. Ticking items there does not touch the real
  aircraft, and the aircraft's E/WD checklist does not tick from it. Use the
  Checklist window above for the real checklists. (The flyPad's "auto-fill
  checklists" Realism setting only auto-marks the tablet's own page.)

  The static cold-and-dark-to-shutdown text checklist and a step-by-step MCDU
  guide are in docs/a380-mcdu-guide.md.

A380 MCDU / MFD Window (open with Input mode, then Shift+M):

  The A380 "MCDU" is the MFD — a cursor-driven display. The window shows one
  thing per line. Actionable lines end with a role word — a button ("PERF,
  button"), an edit field ("CI: 60, edit"), or a menu ("INSERT*, menu"); lines
  with no role word are static text. An empty editable field reads as "blank".
  The page title is announced once when the page changes; the scratchpad /
  message line is announced when it changes.

  Reaching pages — three ways, all keyboard accessible:
   1. "Go to MFD page" combo box (top of the window): lists ALL 20 pages
      grouped by tab (ACTIVE: F-PLN / PERF / FUEL & LOAD / WIND / INIT;
      POSITION: MONITOR / REPORT / NAVAIDS / IRS / GNSS / TIME; SEC INDEX:
      SEC 1-3; DATA: STATUS / WAYPOINT / NAVAID / ROUTE / AIRPORT / PRINTER).
      Pick one to jump straight to it.
   2. The quick page buttons (INIT, F-PLN, PERF, RAD NAV, SEC FPL, ATC COM,
      DIR) with their Alt+letter mnemonics, for the common pages.
   3. The MFD's own top tabs, which appear in the page line list.
  Some pages (e.g. WIND, REPORT, GNSS, TIME, the DATA sub-pages) only open
  once the FMS has the flight plan / data they need — that is the real
  aircraft behaviour.

  Up/Down Arrow   Move through the lines
  Enter           Activate the selected line; if you typed into the scratchpad
                  first, the text is sent to the selected field
                  (A combo box reads "collapsed" when closed; Enter opens it and
                  it reads "expanded", with its choices listed just below.)
  (type)          Type into the scratchpad edit box, then Enter to send
  Backspace       CLR (clears the scratchpad first, then the selected field)
  Ctrl+Up         Previous page (PageUp also works)
  Ctrl+Down       Next page (PageDown also works)
  MCDU (Captain) / First Officer selector chooses which side you are driving.

  Page buttons (mnemonics, Alt+letter):
  Alt+I   INIT       Alt+F   F-PLN      Alt+P   PERF
  Alt+R   RAD NAV    Alt+S   SEC FPL    Alt+A   ATC COM
  Alt+D   DIR        Alt+H   Refresh

  (Page buttons navigate by clicking the MFD's own page-selector menu items, so
  they work whether or not the dropdown is open.)

  Revising the flight plan (on the F-PLN page):
   - Each waypoint line ends with "waypoint" — it is actionable. Enter on it opens
     that waypoint's revision menu (DIR TO, INSERT NEXT WPT, DELETE, HOLD, AIRWAYS,
     OVERFLY). The menu choices then appear below as "option" lines — arrow to one
     and Enter. Disabled choices read "dimmed".
   - A gap in the route reads "DISCONTINUITY, button". To clear it: Enter on that
     line, then arrow to "DELETE *" and Enter — the two legs join.
   - Some revisions stage a TMPY plan: scroll to the bottom and Enter "TMPY INSERT"
     to commit (or "TMPY ERASE" to cancel).

  Loading a SimBrief / company flight plan (on the INIT page, Alt+I):
   1. Arrow to "CPNY F-PLN REQUEST" and Enter to request it (needs your SimBrief
      user ID set in the flyPad). Wait for the scratchpad: "COMPANY F-PLN RECEIVED
      WAITING FOR INSERTION".
   2. The button now reads "RECEIVED CPNY F-PLN" — Enter on it to open a small
      menu, arrow to "INSERT*" and Enter. The whole route loads (flight number,
      from/to, alternate, cruise level, cost index all fill in from SimBrief).
   3. FLT NBR, the destination, and ALTN stay editable afterwards if you want to
      change them — arrow to the field, type, Enter. (Forgetting to type them
      first is fine; SimBrief fills them in on insert.)
   4. Departure runway + SID: Alt+F (F-PLN) then the DEPARTURE button, choose RWY,
      then SID, then "INSERT TMPY *" to commit. (Runway/SID changes use a
      temporary plan you confirm with INSERT TMPY; the SimBrief route itself does
      not.)
   5. Arrival (STAR / approach): on the F-PLN page, page down (Ctrl+Down) to the
      destination airport line at the END of the route, activate it, choose
      ARRIVAL, then pick the approach / STAR / VIA, and INSERT TMPY. (The bottom
      "DEST OMDB" summary line is just a readout — it does not open the revision.)

  Landing-rollout safety call-outs (announced automatically, no key): on approach
  and rollout MSFSBA speaks the A380's runway-overrun protection — "Runway too
  short", "If wet, runway too short", "Max braking" — and "Runway ahead" when you
  taxi toward a runway. These come straight from the aircraft's ROW/ROP and OANS
  systems, the same logic that drives the PFD warnings.


General:
  Escape     Exit active hotkey mode

Notes:
  - There is no FCU knob hardware difference to worry about: the A380 reuses the
    same FCU set/read flow as the A320.
  - Many overhead/pedestal systems are A380-specific (4 engines, Green/Yellow
    hydraulics, 11 fuel tanks with transfer + jettison, PRIM/SEC flight-control
    computers, BTV autobrake, 3 RMP radios). All are in the Sections/Panels tree.
  - This is a development build; some control or readout names may not match
    perfectly. Report anything that reads or sets wrong.
